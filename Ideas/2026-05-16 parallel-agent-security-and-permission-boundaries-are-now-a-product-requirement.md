# Parallel-agent security and permission boundaries are now a product requirement

## Thesis
The highest-leverage users are already thinking about what agents must not access; agent velocity without permission design becomes an interview/security failure mode.

## Why now
Multiple cards discussed parallel agents, security break-in tests, and supply-chain attacks targeting AI developer tooling.

## Next move
Add an agent-permission threat model to Hermes/OMX workflows before increasing autonomy.

## Execution path
- List current tools/files/secrets available to coding agents
- Define default-deny boundaries for browser, shell, git, credentials, and tmux
- Run one red-team pass where a worker tries to exfiltrate or mutate out-of-scope state
