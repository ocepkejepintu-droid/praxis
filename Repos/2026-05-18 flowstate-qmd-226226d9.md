# flowstate-qmd

Source: hermes-atlas
Category: Memory & Context
Author: amanning3390

FlowState-QMD is a local-first memory layer designed to provide coding agents with immediate project context from markdown documentation. It utilizes a SQLite-based vector store and an anticipatory cache that prefetches relevant documents by monitoring active agent session logs. This approach allows agents to access design docs, ADRs, and changelogs before initiating a reactive search loop. The system integrates via an MCP server and supports local execution using GGUF models and node-llama-cpp. Highlights: Anticipatory cache prefetches context by watching agent session logs; Local-first storage using SQLite-vec and GGUF models via node-llama-cpp; MCP-native server with automated configuration for major coding agent clients Anticipatory memory with RAG and vector search — Hermes 2026 Hackathon entry
