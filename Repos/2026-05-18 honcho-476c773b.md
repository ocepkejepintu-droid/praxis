# honcho

Source: hermes-atlas
Category: Memory & Context
Author: plastic-labs

Honcho is a memory infrastructure library designed to help developers build stateful agents that track evolving entities like users, groups, and projects over time. It functions through a background reasoning loop that extracts conclusions from stored messages and events rather than relying solely on simple vector matching. This approach allows agents to query natural-language insights, peer representations, and session context to inform future model calls. The system supports both a managed cloud service and a self-hosted FastAPI server with integrations for MCP, Claude Code, and Hermes. Highlights: Asynchronous reasoning extracts conclusions from conversation history and events; Peer-centric model tracks evolving relationships between users and agents; Provides prompt-ready context via Python and TypeScript SDKs Memory library for building stateful agents — the upstream library that the elkimek/honcho-self-hosted Hermes wrapper builds on.
