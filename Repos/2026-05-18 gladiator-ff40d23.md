# gladiator

Source: hermes-atlas
Category: Multi-Agent & Orchestration
Author: runtimenoteslabs

Gladiator is an autonomous agent arena that pits two AI companies against each other to compete for GitHub stars on identical repositories. The system uses Paperclip for orchestration and Hermes Agent workers to execute tasks according to distinct corporate strategies: growth-focused Blitz versus quality-focused Craft. A real-time dashboard visualizes the competition through scoreboards, task boards, and audit trails. Upon completion, the project demonstrates cross-company skill transfer by merging the rival entities and their learned memories. Highlights: Orchestrates 9 autonomous agents across two competing AI companies; Uses Paperclip and Hermes Agent for task execution and learning; Real-time dashboard visualizes agent tasks, code, and scoreboard metrics Autonomous AI companies competing for GitHub stars — agent-vs-agent arena
