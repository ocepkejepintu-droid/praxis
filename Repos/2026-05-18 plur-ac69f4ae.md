# plur

Source: hermes-atlas
Category: Memory & Context
Author: plur-ai

PLUR is a persistent, local-first memory layer designed to synchronize agent corrections, preferences, and conventions across different tools and sessions. It stores knowledge as plain YAML "engrams" on disk and uses a hybrid BM25 and embedding-based retrieval system to inject relevant context into agent prompts. The system employs an ACT-R activation model where facts strengthen with use and naturally decay over time to prevent context clutter. It integrates directly with MCP-compatible tools, Claude Code, and Hermes Agent to automate the capture and recall of project-specific knowledge. Highlights: Stores memory locally as open-format YAML engrams with zero API costs; Uses ACT-R activation to strengthen relevant facts and decay stale ones; Provides MCP tools for automated learning and hybrid context injection Shared memory layer with open engram YAML format for multi-agent systems
