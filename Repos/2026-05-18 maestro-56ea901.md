# maestro

Source: hermes-atlas
Category: Skills & Skill Registries
Author: ReinaMacCredy

Maestro is a local-first agent harness that provides a structured state layer for the software development lifecycle. It uses a CLI and on-disk state model in a .maestro/ directory to manage specs, tasks, and handoffs without requiring a database or background daemon. By persisting continuation records and evidence rows locally, it allows separate agent sessions to collaborate and resume work exactly where the previous one finished. The system enforces discipline through machine-checked contracts and a plan-approve-execute workflow that gates task completion on witnessed evidence. Highlights: Local-first state management using on-disk files instead of chat history; Structured spec-to-ship workflow with auditable evidence and witness levels; Passive handoff envelopes for seamless task resumption between different agents Harness for long-running AI agents — structured memory, cross-feature learning, plan-approve-execute
