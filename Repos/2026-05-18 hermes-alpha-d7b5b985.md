# hermes-alpha

Source: hermes-atlas
Category: Forks & Derivatives
Author: kaminocorp

Hermes Alpha is an experimental framework designed to test if a stock AI agent can autonomously bootstrap and manage a bug bounty system from first principles. The architecture utilizes a two-agent loop where a persistent 'Overseer' agent builds, deploys, and refines an ephemeral 'Hunter' agent tasked with finding software vulnerabilities. By providing the agent with only a Linux terminal and an identity document rather than custom tools, the project evaluates the self-improvement capabilities of the Hermes model. The system integrates with Fly.io for managed infrastructure and uses Elephantasm for long-term memory streams. Highlights: Autonomous two-agent loop for self-improving vulnerability research; Web-based terminal interface with persistent WebSocket sessions; Managed deployment of sub-agents via Fly.io infrastructure Cloud-deployed Hermes with pre-configured templates and managed infrastructure
